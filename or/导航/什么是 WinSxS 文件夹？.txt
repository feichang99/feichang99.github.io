什么是 WinSxS 文件夹？
WinSxS 的全称是 Windows Side-by-Side。它是 Windows 系统的“核心仓库”和“后悔药”。其主要功能包括：

系统更新备份： 当系统安装更新时，旧版本的系统组件会保留在这里。如果新更新出问题，系统可以借此“卸载更新”或回滚。

功能开/关： 比如你想在控制面板里开启“Hyper-V”或“沙盒”功能，系统就是直接从 WinSxS 里提取文件，不需要你再插 U 盘安装。

系统修复： 当系统文件损坏，运行 sfc /scannow 命令时，系统就是从这里复制完好的文件去修复 C 盘。

💡 为什么它看起来这么大？
WinSxS 里面大量使用了**“硬链接”（Hard Links）技术。很多文件其实只在硬盘上占了一份空间，但同时在 System32 和 WinSxS 文件夹里显示。第三方查杀软件（如 360、腾讯电脑管家等）或者 Windows 资源管理器在计算大小时，会把它们重复计算，所以它实际占用的空间并没有 11G 那么多**。

可以优化清理吗？
可以，但必须使用 Windows 自带的安全工具。 以下是两种最安全、最有效的清理方法：

方法一：使用“磁盘清理”工具（最推荐，安全无脑）
打开 此电脑，右键点击 C 盘，选择 属性。

在弹出窗口中点击 磁盘清理。

关键一步：点击 清理系统文件 按钮（需要管理员权限）。

在列表中勾选 Windows 更新清理（这里通常是占用空间大户），以及其他你想清理的项。

点击 确定，等待系统自动销毁没用的旧更新备份。

方法二：使用命令行强制深度清理（进阶）
如果你想榨干最后一兆空间，可以使用 Windows 自带的部署映像工具（DISM）。

鼠标右键点击屏幕左下角的 Windows 徽标（或按 Win + X），选择 终端（管理员） 或 命令提示符（管理员）。

第一步：分析是否需要清理
输入以下命令并回车：
Dism /Online /Cleanup-Image /AnalyzeComponentStore
系统会告诉你“组件存储的实际大小”以及“是否建议进行清理”。

第二步：执行深度清理
如果提示可以清理，输入以下命令并回车：
Dism /Online /Cleanup-Image /StartComponentCleanup
这个命令会删除所有被新更新替代的旧版本组件。

（可选）终极绝杀：删除所有历史回滚版本
如果你确定最近的系统更新很稳定，绝不需要卸载更新回滚，可以输入：
Dism /Online /Cleanup-Image /StartComponentCleanup /ResetBase
注意：运行这个命令后，你将无法卸载任何已经安装的 Windows 更新。

通过上述方法清理后，11G 的文件夹大概率能瘦身不少。再次提醒，绝对不要用 Shift+Delete 去强删这个文件夹哦！
